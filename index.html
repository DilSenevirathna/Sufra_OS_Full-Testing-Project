Scope

The QA process covered core functionalities including menu management, pricing, ingredient tracking, image handling, and admin interface responsiveness across desktop and mobile platforms.

Test Issues and Status

TC 1: Price changing automatically - Status: Fail. The system unexpectedly alters price values without user input, leading to potential inaccuracies. Recommendation: Implement a validation mechanism to lock price changes until explicitly confirmed by the user.
TC 2: Calories changing automatically - Status: Fail. Calorie values are being modified automatically, which may confuse users. Recommendation: Disable automatic calorie updates and require manual entry or calculation confirmation.
TC 3: Availability time not reflecting after saving changes - Status: Fail. Changes to availability time fail to persist after saving, causing data inconsistency. Recommendation: Debug the save function to ensure all changes are properly stored and reflected.
TC 4: Image not reflecting after saving changes - Status: Fail. Uploaded or edited images do not display after saving, impacting visual data integrity. Recommendation: Verify the image upload and retrieval process to ensure proper file handling and display.
TC 5: Ingredients not reflecting after saving changes - Status: Fail. Edited ingredient details are not updated post-save, leading to outdated information. Recommendation: Review the ingredient update logic to ensure changes are saved and displayed correctly.
TC 6: Edited price not reflecting after saving changes - Status: Fail. Price modifications are not saved or reflected, causing financial discrepancies. Recommendation: Fix the price update pipeline to ensure edits are persisted and visible.
TC 7: Calories not reflecting after saving changes - Status: Fail. Calorie edits are not saved or displayed, affecting nutritional information accuracy. Recommendation: Ensure the calorie update process mirrors the price update fix for consistency.
TC 8: Menu item can add without price - Status: Fail. Allowing menu items to be added without a price violates business rules. Recommendation: Enforce a mandatory price field before saving a new menu item.
TC 9: Cursor incorrectly placed in the price input field during editing - Status: Fail. The cursor placement disrupts the editing experience for price values. Recommendation: Adjust the input field focus logic to place the cursor at the end or selected text as per user expectation.
TC 10: Placeholder sample text in input field should disappear when user start type - Status: Fail. The placeholder text remains visible, confusing users during data entry. Recommendation: Implement a standard input event listener to clear placeholder text on user input.
TC 11: More Ingredients + icon cannot viewable hidden ingredients - Status: Fail. The inability to view hidden ingredients via the + icon limits functionality. Recommendation: Enhance the UI to ensure the + icon is accessible and reveals all ingredients.
TC 12: Restaurant Admin page not mobile responsive - Status: Fail. The admin page does not adapt to mobile screen sizes, hindering usability on smaller devices. Recommendation: Implement responsive design techniques, such as media queries and flexible layouts, to ensure compatibility across mobile platforms.
